04 — Key Distinction

Authentication vs Authorization

🔑

Authentication

WHO are you?

Verifies user identity — the login process.

  • Local username + password
  • LDAP / Active Directory
  • SAML 2.0 SSO (Azure AD, Okta)
  • OAuth 2.0 tokens
  • Multi-Factor Authentication
  • Certificate-based (mTLS)
VS
🛡

Authorization

WHAT can you do?

Determines permissions after identity is verified.

  • ACLs (primary enforcement)
  • Roles (permission sets)
  • Business Rules (gs.hasRole)
  • UI Policies (field control)
  • Data Policies (server-side)
  • Domain Separation
⚠️ Interview Tip: This distinction appears in every ServiceNow CSA exam and job interview. Auth = WHO. Authz = WHAT.

08 — GlideUser API

Key API Methods

user-info.js
// Get current user details
var userId   = gs.getUserID();
var userName = gs.getUserName();
var display  = gs.getUserDisplayName();

// GlideUser object
var user = gs.getUser();
gs.info(user.getEmail());
gs.info(user.getManagerID());
gs.info(user.getDepartmentID());
check-roles.js
// Check role
if (gs.hasRole('itil')) {
  gs.info('Has ITIL access');
}

// Check group membership
var inGroup = gs.getUser().isMemberOf(
  'IT Support L2'
);

// Get all user's groups
var groups = gs.getUser().getMyGroups();
client-api.js
// ⚠️ Client-side = UI only, NOT secure
var name = g_user.userName;
var id   = g_user.userID;

// Role check (admin = true for ALL)
g_user.hasRole('itil');

// Exact check (excludes admin)
g_user.hasRoleExactly('itil');

// Check from list
g_user.hasRoleFromList('itil,admin');
client-pattern.js
// Role-based UI control pattern
function onLoad() {
  if (g_user.hasRole('itil')) {
    g_form.setSectionDisplay(
      'technical', true
    );
  } else {
    g_form.setSectionDisplay(
      'technical', false
    );
  }
}
query-users.js
// Find user by username
var gr = new GlideRecord('sys_user');
gr.addQuery('user_name', 'john.smith');
gr.query();
if (gr.next()) {
  gs.info(gr.getValue('email'));
}
query-groups-roles.js
// Get all members of a group
var gm = new GlideRecord('sys_user_grmember');
gm.addQuery('group.name', 'IT Support');
gm.query();
while (gm.next()) {
  gs.info(gm.getDisplayValue('user'));
}
09 — Insider Knowledge

Pro Developer Tips

01

Groups Over Direct Assignment

Always assign roles to groups. Managing 500 individual role assignments is a nightmare. One group change handles all.

02

hasRoleExactly() for Precision

g_user.hasRole('itil') returns true for admins. Use hasRoleExactly() to exclude admin users.

03

Security Diagnostics Page

Navigate to sys_security_acl_debug.do — test ACL evaluation for any user. Shows exactly which rule grants or denies.

04

Impersonate, Don't Re-Login

Test access via avatar → "Impersonate User." All actions are logged. Much faster than logging out and back in.

05

Debug sys_user_has_role

When access fails — check this table first. Filter by user, add inherited and granted_by columns.

06

Elevated Privileges Safety Net

Keep security_admin elevated. Admins must explicitly activate it each session.

07

Use isMemberOf() Built-in

Don't query sys_user_grmember manually. Use gs.getUser().isMemberOf() — one line, faster.

08

Module Visibility = Roles

Navigation modules check roles. No itil? No Incident module. Check Application Menus. CSA exam favorite!

09

Before Query for Row Security

Need "managers see only their team's data"? Use Before Query Business Rule with current.addQuery().

10

Scoped App Role Naming

Always prefix custom roles: x_company.app_name.role. Never use generic names in Global scope.

11 — Knowledge Check

Test Yourself

0/7
Score
Question 01 Easy
Which table stores user records in ServiceNow?
A sys_user
B sys_user_role
C sys_user_group
D cmn_user
✅ A — sys_user is the core table for all user records. sys_user_role stores role definitions. cmn_user doesn't exist in ServiceNow.
Question 02 Easy
Authentication verifies ___, Authorization determines ___.
A access, identity
B identity, access
C same thing
D client-side, server-side
✅ B — Authentication = WHO (identity). Authorization = WHAT (permissions). Classic interview question!
Question 03 Medium
True or False: Roles assigned to a group are inherited by all members.
A True
B False
✅ True — Core role inheritance mechanism. Check sys_user_has_role where inherited = true.
Question 04 Medium
Admin user runs g_user.hasRole('itil'). What's returned?
A false
B true
C Error
D Depends
✅ B (true) — admin passes ALL role checks. Use hasRoleExactly() to exclude admin override.
Question 05 Medium
Which is NOT a valid ACL operation?
A read
B write
C execute
D modify
✅ D (modify) — Valid: read, write, create, delete, execute. Updates use "write", not "modify".
Question 06 Easy
True or False: Delete user records when employees leave.
A True
B False
✅ False — NEVER delete. Set active=false. Deletion breaks audit trails and compliance.
Question 07 Hard
Sarah sees the Incident module but gets "Access Denied" on records. She has itil via group. Most likely cause?
A Inactive account
B Expired role
C Record/field-level ACL denying
D Needs to elevate itil
✅ C — Table ACL passes (sees module) but record/field ACL denies. Use sys_security_acl_debug.do to diagnose.
